# Tokens Package

The `@armory/tokens` package provides pre-configured token objects for common stablecoins and tokens across multiple chains.

## Installation

```bash
bun add @armory/tokens
```

## Key Exports

### Token Objects

```typescript
import {
  TOKENS,
  getToken,
  getAllTokens,
  getTokensBySymbol,
  getTokensByChain,
  getUSDCTokens,
  getUSDTTokens,
  getDAITokens,
  getWBTCTokens,
} from '@armory/tokens'
```

### Types

```typescript
import type {
  TokenConfig,
  CustomToken,
} from '@armory/tokens'
```

## Usage

### Import Individual Tokens

```typescript
import { TOKENS } from '@armory/tokens';

// Use with facilitator
const result = await verifyPayment(payload, {
  token: TOKENS.USDC_BASE,
  // ... other options
});
```

### Import All Tokens

```typescript
import { TOKENS } from '@armory/tokens';

// Access any token
const token = TOKENS.USDC_ARBITRUM;
```

### Query Tokens

```typescript
import {
  getToken,
  getTokensBySymbol,
  getTokensByChain,
  getUSDCTokens,
} from '@armory/tokens';

// Get token by chain and address
const token = getToken(8453, '0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913');

// Get all tokens for a symbol
const allUsdc = getTokensBySymbol('USDC');
// Returns: [USDC_ETHEREUM, USDC_BASE, USDC_OPTIMISM, ...]

// Get all tokens on a chain
const baseTokens = getTokensByChain(8453);
// Returns: [USDC_BASE, USDT_BASE, DAI_BASE, WBTC_BASE]

// Get USDC tokens
const usdcTokens = getUSDCTokens();
```

### Token Configuration

Each token object includes:

```typescript
interface TokenConfig {
  symbol: string;              // Token symbol (e.g., "USDC")
  name: string;                // Token name (e.g., "USD Coin")
  version: string;             // EIP-3009 version
  contractAddress: `0x${string}`; // Contract address
  chainId: number;             // Chain ID
  decimals?: number;           // Token decimals
  rpcUrl?: string;             // RPC URL for the chain
  domainName: string;          // EIP-712 domain name
  domainVersion: string;       // EIP-712 domain version
}
```

## Integration Examples

### With Viem Client

```typescript
import { createX402Client } from '@armory/client-viem';
import { TOKENS } from '@armory/tokens';

const client = createX402Client({
  wallet: { type: 'account', account },
  defaultToken: TOKENS.USDC_BASE,
});
```

### With Ethers Client

```typescript
import { createX402Client } from '@armory/client-ethers';
import { TOKENS } from '@armory/tokens';

const client = createX402Client({
  signer,
  defaultToken: TOKENS.USDC_ETHEREUM,
});
```

### With Middleware

```typescript
import { paymentMiddleware } from '@armory/middleware';
import { TOKENS } from '@armory/tokens';

app.use(paymentMiddleware({
  facilitatorUrl: 'http://localhost:3000',
  settlementMode: 'async',
  token: TOKENS.USDC_BASE,
}));
```

### With Facilitator

```typescript
import { verifyPayment } from '@armory/facilitator';
import { TOKENS } from '@armory/tokens';

const result = await verifyPayment(payload, {
  walletClient,
  publicClient,
  nonceTracker,
  token: TOKENS.USDC_BASE,
});
```

## Multi-Chain Support

```typescript
import { getUSDCTokens } from '@armory/tokens';

const usdcTokens = getUSDCTokens();

// Accept payments across all chains
for (const token of usdcTokens) {
  console.log(`Accepting ${token.symbol} on chain ${token.chainId}`);
}
```

## Custom Token Registration

You can also register custom tokens alongside the pre-configured ones:

```typescript
import { registerToken } from '@armory/base';

const myToken = {
  symbol: 'MYTOKEN',
  name: 'My Custom Token',
  version: '1',
  contractAddress: '0x...',
  chainId: 8453,
  decimals: 18,
};

registerToken(myToken);
```
