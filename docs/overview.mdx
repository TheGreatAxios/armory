# Overview

Armory is a modular payment protocol for decentralized commerce, built on EIP-3009 (transferWithAuthorization).

## How It Works

1. **Client** creates an EIP-3009 authorization (off-chain signature)
2. **Client** sends request with `PAYMENT-SIGNATURE` header
3. **Server** verifies the signature via a facilitator
4. **Facilitator** settles the payment on-chain
5. **Client** receives the API response

## Packages

### Core

- **@armory/base** - Protocol types, encoding, EIP-712, network configs

### Server

- **@armory/facilitator** - Payment verification and settlement
- **@armory/middleware** - HTTP middleware for Express, Hono, Bun, Elysia

### Client

- **@armory/client-viem** - Viem payment client
- **@armory/client-ethers** - Ethers.js v6 payment client
- **@armory/client-web3** - Web3.js payment client

### Utilities

- **@armory/tokens** - Pre-configured token objects

## Protocol

Armory uses EIP-3009 `transferWithAuthorization`, allowing payers to create off-chain authorizations that payees can execute on-chain. This enables:

- **Gasless payments for payers** - The facilitator pays gas
- **Replay protection** - Nonces and timestamps
- **Standard compliance** - Works with any EIP-3009 token

## Supported Networks
