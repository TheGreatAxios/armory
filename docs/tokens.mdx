## Using Tokens

### By Symbol (Recommended)

```typescript
import { armoryPay } from '@armory/client-viem';

// Case-insensitive
await armoryPay(wallet, url, 'base', 'usdc');
await armoryPay(wallet, url, 'base', 'USDC');
await armoryPay(wallet, url, 'base', 'UsDc');
```

### By Contract Address

```typescript
// Use any ERC-20 token by its address
await armoryPay(wallet, url, 'base', '0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913');
```

### By CAIP Asset ID

```typescript
// Standards-compliant format
await armoryPay(wallet, url, 'base', 'eip155:8453/erc20:0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913');
```

## Adding Your Own Token

### Option 1: GitHub PR (Recommended)

Contribute your token to the official Armory registry so everyone can use it!

#### Step 1: Fork the Repository

```bash
git clone https://github.com/YOUR_USERNAME/armory.git
cd armory
```

#### Step 2: Add Your Token

Edit `packages/tokens/src/index.ts`:

```typescript
export const MY_TOKEN: TokenConfig = {
  symbol: "MYTOKEN",              // Token symbol (uppercase)
  name: "My Token Name",          // Full token name
  version: "1",                   // EIP-712 domain version
  contractAddress: "0x...",       // Contract address (with 0x prefix)
  chainId: 8453,                  // Network chain ID
  decimals: 18,                   // Token decimals
};

// Add to TOKENS object
export const TOKENS = {
  // ... existing tokens
  MY_TOKEN,
} as const;

// Add to registry initialization
Object.values(TOKENS).forEach(registerToken);
```

#### Step 3: Test Your Token

```bash
bun test
```

#### Step 4: Submit PR

```bash
git add packages/tokens/src/index.ts
git commit -m "Add MYTOKEN token"
git push origin main
# Then open a PR on GitHub
```

#### PR Template

```markdown
## Token Addition

**Token Symbol:** MYTOKEN
**Token Name:** My Token Name
**Network:** Base (Chain ID: 8453)
**Contract Address:** 0x...

### Verification

- [x] Token contract verified on Etherscan
- [x] Contract follows ERC-20 standard
- [x] Tested with `armoryPay` function
- [x] Added to TOKENS object
- [x] Registered in initialization

### Additional Notes

<!-- Any additional information about the token -->
```

### Option 2: Runtime Registration

Register tokens at runtime without modifying the codebase:

```typescript
import { registerToken } from '@armory/core';

// Register your custom token
registerToken({
  symbol: 'MYTOKEN',
  name: 'My Token',
  version: '1',
  contractAddress: '0x...',
  chainId: 8453,
  decimals: 18,
});

// Now you can use it by symbol!
await armoryPay(wallet, url, 'base', 'mytoken');
```

### Option 3: Pass Directly

Use a token without registering it:

```typescript
await armoryPay(wallet, url, 'base', {
  symbol: 'MYTOKEN',
  name: 'My Token',
  version: '1',
  contractAddress: '0x...',
  chainId: 8453,
  decimals: 18,
});
```

## Token Requirements

For a token to be added to the official registry:

1. **ERC-20 Compliance**: Must follow the ERC-20 standard
2. **Verified Contract**: Contract must be verified on Etherscan/block explorer
3. **Mainnet or Testnet**: Must be deployed on a supported network
4. **Decimals**: Accurate decimal count for the token
5. **EIP-712 Support**: Must support EIP-712 typed data signing

## Looking Up Token Info

### Etherscan

1. Go to [etherscan.io](https://etherscan.io)
2. Search for your token address
3. Check the "Contract" tab for verification
4. Note the `decimals` value from the Read Contract section

### BaseScan

1. Go to [basescan.org](https://basescan.org)
2. Same process as Etherscan
